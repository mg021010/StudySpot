1. 프로젝트 배경 및 문제 정의

Context & Problem
😫 핵심 페인 포인트
  • 분위기 매칭 실패: 정적 학습 공간 내 소음 유입으로 인한 몰입 방해 및 갈등 발생
  • 정보 신뢰도 결여: 기존 PIR 방식은 고정 학습자를 감지하지 못해 오판하는 치명적 한계
  • 활동성 정보 부재: 현행 앱의 수치형 혼잡도는 공간의 '정체성'을 설명하지 못함
🚀 CampusSpot의 가치

단순 수치를 넘어 사용자의 현재 목적과 공간의 정체성을 정교하게 매칭합니다.

"공간의 성격은 수치가 아니라 '음향적 정체성'으로 재정의되어야 합니다."

4. 시스템 아키텍처

Tech Stack
📦 Edge Layer
  • Raspberry Pi 컴퓨팅
  • INMP441 (I2S)
  • PIR / BLE Sniffing
  • C++ 20 드라이버
📡 Platform Layer
  • MQTT 초경량 메시징
  • JSON 표준 페이로드
  • Python 분석 엔진
  • InfluxDB 시계열 저장
📱 App Layer
  • 공간 맞춤 추천 알고리즘
  • 카테고리 필터링
  • 사용 패턴 리포트
Flow: Edge Node ➜ MQTT ➜ Server ➜ InfluxDB ➜ End User App
