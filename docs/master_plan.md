# CampusSpot: 공간 음향 정체성 및 하이브리드 점유 분석 기반 학습 공간 큐레이션 시스템

## 1. 아이디어 배경 및 문제 제기 (Why?)

대학생이나 직장인들은 공부나 업무를 위해 카페, 도서관, 학생 식당, 라운지 등 다양한 공간을 찾습니다. 하지만 공간을 선택할 때마다 다음과 같은 불편함을 겪습니다.

*   **"자리 있나?" (점유 상태의 불확실성):** 무거운 짐을 들고 해당 장소까지 갔는데 자리가 꽉 차 있어 헛걸음을 하는 경우가 많습니다.
*   **"여기 왜 이렇게 시끄러워?" (공간 성격의 불일치):** 자리가 있어서 앉았는데, 나는 조용히 집중하고 싶었지만 옆 테이블에서 조별 과제를 하며 큰 소리로 대화하거나, 반대로 나는 화상 회의를 해야 하는데 공간이 너무 조용해서 눈치가 보이는 상황이 발생합니다.
*   **단순한 '혼잡도' 정보의 한계:** 기존의 도서관 좌석 시스템이나 지도 앱의 혼잡도 정보는 단순히 '사람이 많다/적다'만 알려줄 뿐, 사람들이 그 공간에서 '어떤 형태의 활동(조용히 공부 vs 활발한 토론)'을 하고 있는지 공간의 '분위기'는 전혀 알려주지 못합니다.

## 2. 해결책 제안: CampusSpot (What?)

CampusSpot은 이러한 문제를 해결하기 위해, 단순한 빈자리 찾기를 넘어 **공간의 '현재 분위기(음향적 정체성)'와 '실제 밀집도'를 분석하여 사용자에게 필요한 공간을 매칭해주는 시스템**입니다.

### 핵심 차별점 1: '점수'가 아닌 '카테고리'를 통한 공간 성격 정의
공간이 '좋다/나쁘다'를 숫자로 평가하는 방식을 버렸습니다. 대신, 마이크 센서를 통해 수집한 소리의 주파수와 볼륨 특성을 실시간으로 분석하여 공간의 현재 상태를 4가지 직관적인 카테고리로 분류합니다.

1.  🤫 **Deep Focus (절대 정숙):** 도서관 열람실처럼 백색소음조차 적은 극도의 집중이 가능한 공간.
2.  🌬️ **White Noise (백색소음):** 공조기나 일정한 기계음이 깔려 있어 타인의 작은 소음이 마스킹되는 공간.
3.  🎵 **BGM (배경음악):** 카페처럼 음악이 흐르고 적당한 백그라운드 소음이 있는 공간.
4.  🗣️ **Active Discussion (대화/토론):** 사람의 목소리가 주를 이루며 자유롭게 대화와 팀플이 가능한 활기찬 공간.

사용자는 앱을 켜고 "나는 오늘 팀플을 해야 하니까 'Active Discussion' 카테고리의 빈 공간을 찾아줘"라고 직관적으로 선택할 수 있습니다.

### 핵심 차별점 2: 하이브리드 센서 융합을 통한 정확한 밀집도 파악 (How?)
단일 센서의 약점을 보완하기 위해 두 가지 다른 방식의 기술을 융합합니다.

*   **PIR (적외선 인체 감지) 센서:** 즉각적인 '움직임'을 잘 감지합니다. 단, 사람이 책상에 가만히 앉아 집중할 때는 빈 공간으로 오해하는 치명적인 약점이 있습니다.
*   **Bluetooth / Wi-Fi 스니핑:** 공간 내에 있는 스마트 기기(스마트폰, 노트북 등)의 무선 신호(MAC 주소)를 세어 대략적인 인원 규모를 파악합니다. 단, 기기가 없는 사람을 놓치거나 공간 밖에 있는 사람의 신호까지 잡는 노이즈가 있습니다.

**[상호 보완 로직 예시]**
"블루투스 신호는 수십 개가 잡히는데(사람이 많음), PIR 센서에는 움직임이 전혀 잡히지 않는다면?"
👉 기존 시스템은 "움직임이 없으니 빈 공간이다"라고 오판하겠지만, CampusSpot은 **"사람이 가득 차 있지만 모두가 가만히 앉아 고도로 집중하고 있는 정적인 밀집 상태"**로 정확하게 해석합니다.

## 3. 시스템 아키텍처 및 구현 기술

*   **Edge Node (하드웨어 단말 - Raspberry Pi 등):**
    *   **마이크 (INMP441):** 소리를 수집하고 기기 내에서 즉시 고속 푸리에 변환(FFT) 처리를 거쳐 주파수 특성을 추출. (개인정보 보호를 위해 실제 음성 내용은 저장/전송하지 않고 소리의 '패턴 데이터'만 분석)
    *   **PIR 센서:** 움직임 빈도 측정.
    *   **무선 랜카드:** 주기적인 Bluetooth/Wi-Fi 패킷 스니핑으로 주변 기기 수 카운트.
    *   **자체 연산:** 수집된 데이터를 기기 내에서 `AcousticCategorizer` 및 `OccupancyFusion` 로직을 통해 직접 가공.
*   **통신 및 서버:**
    *   Edge Node에서 최종 가공된 **경량 데이터(공간 상태, 밀집도 등)**를 MQTT 프로토콜을 통해 중앙 클라우드/웹 서버로 직접 전송.
    *   중앙 서버는 수집된 각 노드의 상태를 데이터베이스에 저장하고 사용자 웹 인터페이스에 실시간으로 전시.

## 4. 확장 가능성 및 미래 비전 (Scalability)

현재의 분산형 엣지 아키텍처는 향후 완벽한 **'초개인화(Hyper-Personalized) 공간 큐레이션 플랫폼'**으로 확장하기에 최적의 구조입니다.

1.  **플러그 앤 플레이 확장:** 새로운 공간에 노드를 추가하는 것만으로 시스템 전체의 커버리지가 즉각 확장됩니다.
2.  **환경 정보의 확장 (선택의 폭 극대화):**
    *   향후 조도 센서, 이산화탄소(CO2) 센서 등을 추가하여 "자연광이 잘 드는 창가 자리", "환기가 잘 되어 공기가 쾌적한 공간" 등 더욱 미세한 환경 필터링 옵션을 제공할 수 있습니다.
3.  **개인화 및 행동 데이터 학습 (AI 큐레이션):**
    *   사용자의 사용 패턴(어떤 환경에서 가장 오래 머물렀는지)과 일정(캘린더 앱 연동)을 학습하여 맞춤형 공간을 제안하는 스마트 비서로 진화합니다.

# CampusSpot 마스터 플랜 로드맵

### Phase 1: 하드웨어 및 엣지 드라이버 구현 (Completed)
1.  **공통 인터페이스 확립:** 센서 관리를 위한 C++ 기반 `BaseSensor` 클래스 구조 확립.
2.  **드라이버 개발:** DHT11, PIR, Sound, BLE Sniffer 연동 완료.

### Phase 2: 엣지 컴퓨팅 로직 구현 (Completed)
1.  **AcousticCategorizer:** 온디바이스 소리 특성 4종 분류 로직 구현 완료.
2.  **OccupancyFusion:** 온디바이스 PIR + BLE 하이브리드 점유 판별 로직 구현 완료.
3.  **DataAggregator:** 기기 내 데이터 통합 및 전송용 페이로드 생성 기능 구현 완료.

### Phase 3: 분산 통신망 및 MQTT 연동 (In Progress)
1.  **MQTT 클라이언트 구현:** 각 노드에서 서버로 직접 데이터를 쏘기 위한 Paho MQTT C++ 클라이언트 구축.
2.  **데이터 규격화:** 서버 전송용 JSON 페이로드(공간 ID, 상태, 밀집도 포함) 최종 확정.
3.  **예외 처리:** 네트워크 단절 시 데이터 로컬 보관 및 재연결 로직 설계.

### Phase 4: 서버 사이드 및 실시간 시각화 (Upcoming)
1.  **MQTT Broker 및 Backend:** 노드들의 데이터를 수집할 브로커 설정 및 API 서버 구축.
2.  **사용자 웹 인터페이스:** 지도 기반 실시간 공간 상태 전시 페이지 개발.
3.  **실무 튜닝:** 실제 설치 환경에서의 임계값 최적화.
