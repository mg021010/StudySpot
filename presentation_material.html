Slide 1. 공간 Curation: 단순 혼잡도를 넘어 목적 매칭으로

// PARADIGM SHIFT IN STUDY SPACE FINDER

기존의 공간 찾기 앱들은 단순히 "현재 이용자 수" 또는 "좌석 점유율"만을 전달하는 한계가 있었습니다. 정작 학생들은 "조용한 정밀 집중이 가능한지", "노트북 타건 소리가 허용되는지", 혹은 "자유로운 대화식 토론이 가능한 분위기인지"를 알고 싶어 합니다.

CampusSpot은 음향적 분위기(Acoustic Identity)와 정적 밀집(Static Occupancy)을 다중 융합 진단하여, 사용자가 원하는 학습 상태를 실시간으로 조화롭게 매칭(Curation)해 줍니다.

Slide 2. 실시간 데이터 파이프라인 (Data Pipeline)

// SYSTEM ARCHITECTURE & SYNC PIPELINE

디바이스가 감지한 원시 센서 데이터가 새로고침 없이 대시보드에 즉각 반영되기까지의 전체 흐름입니다. Websocket을 지원하는 Firebase Realtime Database 동기화 기술을 사용하여 1초 미만의 갱신 지연(Latency)을 보장합니다.

분류 핵심 기술 / 컴포넌트 역할 및 특징
1. 엣지 레이어 라즈베리파이 센서 노드 (마이크/BLE/PIR) 원시 전압값 및 신호 수치 측정 후 데시벨/기기 수 가공
2. 브릿지 레이어 MQTT Broker + mqtt_to_firebase.py 경량 패킷 MQTT 수신 후 파싱하여 Cloud HTTPS PUT 전송
3. 데이터 클라우드 Firebase Realtime DB 웹소켓 프로토콜을 통하여 등록된 모든 Frontend 클라이언트에 즉시 브로드캐스트
4. 사용자 화면 웹 대시보드 (demo_dashboard.html) 데이터 이벤트 수신 시 즉시 실시간 차트 업데이트 및 순위 정렬 갱신
🎙️ Slide 2 발표 대본 (Click to toggle)
"저희 데이터 파이프라인의 핵심은 속도와 안정성입니다. 라즈베리파이 엣지 노드가 MQTT 통신을 통해 로컬 브로커로 데이터를 보내면, 저희가 개발한 파이썬 브릿지 프로그램이 클라우드 서버인 Firebase 데이터베이스로 변환하여 덮어씁니다. 대시보드 웹앱은 Firebase를 실시간 구독하고 있어, 사용자가 브라우저를 새로고침하지 않아도 1초 이내에 차트와 잔여석 색상이 변하는 다이나믹한 실시간 시연을 완성해 냈습니다."

Slide 3. 1단계: 실측 데이터의 센싱 기준 (Thresholds)

// SENSOR THRESHOLDS & HYBRID DETECTION SYSTEM

🔊 1. 음향 분위기 판정 (데시벨 변환식 기반)

공식: $dB = 20 \log_{10}(\text{RMS}) + 100$

데시벨 범위 ($dB$) 음향 분위기 판정 (Acoustic Category) 현장 소음 성격 예시
$dB < 30.0$ 🤫 Deep Focus (정막) 숨소리와 가벼운 필기구 마찰만 들리는 초정숙 공간
$30.0 \le dB < 38.0$ 🌬️ White Noise (백색소음) 도서관 및 조용한 노트북 타건음 허용 분위기
$38.0 \le dB < 45.0$ 🎵 BGM (배경음악) 가벼운 배경음악이 있는 모던한 카페 수준
$dB \ge 45.0$ 🗣️ Active Discussion (토론) 조별 미팅, 팀프로젝트 대화 및 발표 공간

📶 2. 하이브리드 점유 판정 (BLE 디바이스 $N$ & PIR 타이머 $C$)

BLE 탐색 수 ($N$) PIR 움직임 쿨다운 ($C$) 점유 상태 판정 결과 동적 의미 분석
$N < 0.5$ $C = 0$ (움직임 없음) ⚪ Vacant (비어있음) 유휴 상태인 열린 공간
$N < 3.5$ $C = 0$ (움직임 없음) 🟢 Static Low (매우 여유) 소수 이용자가 고요히 집중 공부 중
$N < 3.5$ $C > 0$ (움직임 있음) 🟢 Dynamic Low (여유) 소수 이용자가 가벼운 작업 진행 중
$N \ge 3.5$ $C = 0$ (움직임 없음) 🟡 Static High (정적 밀집) 다수 이용자가 숨죽여 초집중하는 자습 상태
$N \ge 3.5$ $C > 0$ (움직임 있음) 🔴 Dynamic High (혼잡) 사람이 많고 활발히 대화/토론하는 다이나믹한 상태
🎙️ Slide 3 발표 대본 (Click to toggle)
"수집된 센서 수치들은 다음과 같은 논리 기준을 가집니다. 마이크 전압값은 로그 변환을 거쳐 30데시벨 미만의 초정막 상태부터 45데시벨 이상의 토론 상태까지 분류됩니다. 특히 점유 상태 판정에 공을 들였습니다. 일반 PIR 센서는 사용자가 꼼짝 않고 공부하면 오인식되어 빈방 처리를 해버립니다. 저희는 주변의 블루투스 신호 대수(N)와 10초 쿨다운을 결합한 알고리즘을 얹어, 블루투스 기기가 3.5대 이상 탐색되는데 정지 상태라면 이를 'Static High', 즉 '조용하게 초집중 중인 정적 밀집 상태'로 오차 없이 감지해 냅니다."

Slide 4. 2단계: 종합 매칭 공식 상세 분석 (Scoring Model)

// ALGORITHM VARIABLES & WEIGHED MATRIX

공간 추천 적합도 점수는 기본 점수 50점에 소음 가중치($W_{acoustic}$), 혼잡도 가중치($W_{occupancy}$)를 융합 연산하고, 학습 피로도를 유발하는 비쾌적 온도 범위에 해당하는 경우 페널티($P_{temp}$) 감점 5점을 삭감하여 계산합니다.

$$\mathbf{SCORE = S_{base} + W_{acoustic} + W_{occupancy} - P_{temp}}$$
Base constant
기본 신뢰 점수 ($S_{base}$)
50점 고정 부여. 가중치 합산과 페널티 감산의 중심점 기준선이 되며, 최종 매칭 점수가 극단적으로 흔들리지 않도록 방어하는 버팀목입니다.
Acoustic Weight
음향 가중치 ($W_{acoustic}$)
사용자의 목적(자습/노트북/토론)에 맞는 음향 분위기별 점수를 매핑합니다. 최대 +40점에서 최하 -40점까지 부여하여 맞지 않는 시끄러운 분위기를 걸러냅니다.
Occupancy Weight
점유 가중치 ($W_{occupancy}$)
정적/동적 혼잡도에 매칭을 부여합니다. 자습 시에는 한적함을 선호하므로 +18점을, 혼잡한 토론 분위기에는 -30점의 큰 감점을 부여합니다.

🌡️ 학습 능률에 따른 온도 페널티 ($P_{temp}$) 로직

인간 공학 및 학습 능률 연구 보고에 따르면 공부 시 가장 집중이 잘 되고 쾌적함을 유지하는 온도는 $21.0^\circ\text{C} \sim 24.5^\circ\text{C}$ 사이입니다.

  • $P_{temp} = 0\text{점}$ (감점 없음): 최적 온도 범위 ($21.0^\circ\text{C} \le T \le 24.5^\circ\text{C}$) 내부인 경우
  • $P_{temp} = 5\text{점}$ 감점 ($S - 5$): 최적 범위 이탈 ($T < 21.0^\circ\text{C}$ 또는 $T > 24.5^\circ\text{C}$)로 다소 덥거나 서늘한 쾌적도 저하 상태인 경우
🎙️ Slide 4 발표 대본 (Click to toggle)
"저희 적합도 스코어 산출 모델은 기본 점수 50점에 소음 가중치, 혼잡도 가중치를 곱하는 게 아닌 '합산(Sum)'하고, 실내 온도가 최적 범위를 벗어나면 5점을 깎아버리는 페널티 차감 연산 구조를 가집니다. 공부 능률에 최적인 21도와 24.5도 사이의 쾌적한 공간을 사용자에게 추천하기 위해 온도 차감 변수를 두었고, 최종 스코어는 화면 표시 상의 신뢰도를 위해 최소 10%에서 최대 98% 범위 내로 고정하도록 바인딩 필터를 마지막 단계에 걸어 두었습니다."

Slide 5. 실제 수치 대입 및 시나리오 연산 사례 (ML415)

// REAL-TIME DATA COMPUTATION & MATRIX MATCHING

실제 기동 중인 공학관 4층 ML415 연구실 자습 구역의 센싱 데이터를 매칭 공식에 대입한 정합성 검증 예시입니다.

센서 구분 실측 데이터 값 분류 판정 상태 자습(Study) 필터 가중치
🌡️ 실내 온도 $25.2^\circ\text{C}$ 비쾌적 상태 (덥고 건조함) 페널티 감점: $P_{temp} = 5$
🔊 소음 레벨 $32.5\text{ dB}$ White Noise (백색소음) 분위기 보너스 가중치: $W_{acoustic} = +15$
📶 BLE 대수 $5.4\text{ 대}$ (기기 감지) Static High (정적 밀집 자습) 중립 가중치: $W_{occupancy} = 0$
🏃 PIR 감지 최근 10초간 무검출 ($C=0$)

🏁 최종 스코어 도출 식

$$S = 50\,(Base) + 15\,(W_{acoustic}) + 0\,(W_{occupancy}) - 5\,(P_{temp}) = 60\text{점}$$

최종 연산 결과: 적합도 **$60\%$** 도출 ➡️ 대시보드 카드에 **[보통] 등급** (45점 이상 75점 미만 영역)으로 매핑 노출 및 실시간 추천 목록 정렬 반영.

🎙️ Slide 5 발표 대본 (Click to toggle)
"실측 데이터를 활용해 자습 구역인 ML415의 적합도를 구해 보겠습니다. 실시간으로 측정된 온도는 25.2도로 다소 높고, 소음은 32.5데시벨의 백색소음 상태이며, 5.4대의 블루투스 무선 기기가 잡히지만 공부 중이라 움직임이 없는 정적 밀집 상태입니다. 사용자가 자습 모드를 선택했다면 백색소음 보너스 +15점과 정적 밀집 중립 0점을 얻습니다. 여기에 온도가 높아 집중을 저해하므로 페널티 5점이 감점되어 기본 50점에 가감 연산을 거쳐 최종 60% 점수가 계산됩니다. 이 점수는 '보통' 추천 등급에 해당하며, 실시간 렌더링 카드에 직관적으로 표기됩니다."

Slide 6. 핵심 예상 질의응답 (Q&A Strategy)

// PROJECT DEFENSE & TECHNICAL BACKUP

Q1. PIR 센서는 사람이 가만히 있으면 측정이 멈추지 않나요?

A1. 네, 열적 움직임을 잡는 PIR 특성상 가만히 앉아 공부하면 움직임이 0으로 잡힙니다. 저희는 이를 대비해 (1) 감지 후 10초간 활성을 누적시키는 쿨다운 타이머 소프트웨어를 구현했고, (2) 블루투스 디바이스 스캔 수(N)를 크로스체크하여 움직임 센서가 0이 되더라도 디바이스 수가 유지된다면 '정적 밀집(Static High)' 집중 상태로 판단하도록 보완 알고리즘을 얹어 오차를 해결했습니다.

Q2. 쾌적 학습 온도 범위(21.0°C ~ 24.5°C) 설정의 기준이 있나요?

A2. 산업통상자원부 에너지 절약 연구 자료 및 교육공학 학회지의 학습 효율성 측정 지표 논문을 참조했습니다. 인간은 21도 미만에서는 집중을 위한 열 생산 에너지를 소모하며, 24.5도를 넘어서면 뇌 혈류 이완으로 인해 졸음과 피로가 빠르게 누적되는 것으로 입증되었습니다. 따라서 해당 최적 영역을 이탈하면 피로 보정치로 감점 5점을 부여해 실제 쾌적도를 반영했습니다.

Q3. 데시벨 변환 공식 상의 오프셋 상수 (+100) 값의 근거는 무엇입니까?

A3. 마이크 아날로그 센서로부터 들어오는 원시 전압 제곱평균제곱근(RMS) 값은 $0.0001$ 등의 소수점 자리수로 나타납니다. 이 값을 사람들이 공간 분석 시 직관적으로 이해할 수 있는 데시벨 단위 스케일(30dB ~ 60dB)로 스케일링 및 정규화하기 위해, 실제 실험실 환경의 정밀 소음측정기와 교차 대조 캘리브레이션 연산을 수행한 결과 오프셋 상수 $+100$을 도출하여 변환 공식에 대입했습니다.
