# 🏫 CampusSpot: 실시간 공간 매칭 알고리즘 PPT 슬라이드 가이드

이 문서는 **CampusSpot**의 핵심 매칭 점수 산출 공식의 각 구성 요소를 심사위원이나 청중에게 수학적/코드 레벨에서 상세하게 입증하고 설명할 수 있도록 설계한 **백업 슬라이드용 상세 명세서**입니다.

---

# 🖥️ [슬라이드 디자인 시안] 알고리즘 핵심 요소 상세 분석

## 🧮 종합 매칭 스코어 산출 공식
$$\mathbf{SCORE = S_{base} + W_{acoustic} + W_{occupancy} - P_{temp}}$$

> [!IMPORTANT]
> **💡 수식 부호 알림**
> 사용자가 쾌적한 온도에 있을 때는 감점이 없고($P_{temp} = 0$), 최적 온도 범위를 벗어나 춥거나 더울 때 쾌적도가 떨어지므로 **온도 페널티를 감점($- P_{temp}$)**하는 구조로 설계되어 있습니다.

---

## 1. $S_{base}$ : 기본 상수 점수 (Base Score)
*   **할당 수치**: **$50\text{점}$**
*   **역할 및 의의**: 
    *   모든 학습 공간이 기본적으로 갖는 추천 신뢰도의 시작점입니다.
    *   환경이 극단적으로 좋거나 나쁠 때 가중치와 페널티가 합산 및 차감되어 최종 점수가 신뢰성 높은 범위($10\% \sim 98\%$) 내로 자연스럽게 수렴하도록 돕는 수학적 기준선(Baseline) 역할을 합니다.

---

## 2. $W_{acoustic}$ : 음향 분위기 가중치 (Acoustic Weight)
*   **입력 데이터**: 마이크 센서로부터 측정된 실시간 데시벨 값 ($dB = 20 \log_{10}(\text{RMS}) + 100$)
*   **분위기 판정 단계**:

| 데시벨 구간 | 음향 분위기 판정 (Category) | 공간 특성 설명 |
| :--- | :--- | :--- |
| **$dB < 30.0$** | **🤫 Deep Focus (정막)** | 속삭임조차 없는 도서관 수준의 정숙함 |
| **$30.0 \le dB < 38.0$** | **🌬️ White Noise (백색소음)** | 가벼운 마우스 클릭, 책장 넘기는 소리 허용 |
| **$38.0 \le dB < 45.0$** | **🎵 BGM (배경음악)** | 카페 음악이나 잔잔한 생활 소음 수준 |
| **$dB \ge 45.0$** | **🗣️ Active Discussion (토론)** | 활발한 대화 및 조별 과제 진행 수준 |

*   **선택 필터(목적)별 가중치 테이블 ($W_{acoustic}$)**:

```
                  [ 🤫 몰입 자습 ]       [ 💻 노트북 작업 ]     [ 🗣️ 조별 토론 ]
Deep Focus     :      +30                   +20                   -40
White Noise    :      +15                   +30                   -10
BGM            :      -20                   +15                   +25
Discussion     :      -40                   -15                   +40
```
*(예: 조용한 자습을 원할 때 토론 소음이 들리면 40점이 감점되고, 토론을 원할 때 토론 분위기라면 40점이 보너스로 합산됩니다.)*

---

## 3. $W_{occupancy}$ : 하이브리드 점유 가중치 (Occupancy Weight)
*   **입력 데이터**: 
    1.  $N$: BLE(블루투스 디바이스) 실시간 스캔 개수 (실수형 변환)
    2.  $C$: PIR 인체 감지 센서 10초 쿨다운 타이머 ($C > 0$: 움직임 있음, $C = 0$: 움직임 없음)
*   **점유 및 밀집도 판정 단계**:

| BLE 수 ($N$) | PIR 타이머 ($C$) | 판정 상태 (Occupancy) | 공간 점유 성격 |
| :--- | :--- | :--- | :--- |
| $N < 0.5$ | $C = 0$ | **⚪ Vacant (비어있음)** | 아무도 이용하지 않는 유휴 공간 |
| $N < 3.5$ | $C = 0$ | **🟢 Static Low (매우 여유)** | 극소수 인원이 조용히 자습 중 |
| $N < 3.5$ | $C > 0$ | **🟢 Dynamic Low (여유)** | 소수 인원이 노트북 작업이나 이동 중 |
| **$N \ge 3.5$** | **$C = 0$** | **🟡 Static High (정적 밀집)** | **많은 사람이 있지만 모두 조용한 집중 상태** |
| $N \ge 3.5$ | $C > 0$ | **🔴 Dynamic High (혼잡)** | 다수 인원이 조별 모임으로 활발히 활동 중 |

*   **선택 필터(목적)별 가중치 테이블 ($W_{occupancy}$)**:

```
                  [ 🤫 몰입 자습 ]       [ 💻 노트북 작업 ]     [ 🗣️ 조별 토론 ]
Static Low     :      +18                   +10                     0
Vacant         :      +18                    +5                     0
Dynamic Low    :       +5                   +18                    +8
Static High    :        0                     0                     0
Dynamic High   :      -30                     0                    +8
```

---

## 4. $P_{temp}$ : 학습 최적 온도 페널티 (Temperature Penalty)
*   **입력 데이터**: 실내 온도 측정값 ($T$, $^\circ\text{C}$)
*   **학습 최적 온도 구간**: **$21.0^\circ\text{C} \le T \le 24.5^\circ\text{C}$**
*   **페널티 적용 기준 ($P_{temp}$)**:
    *   **$0\text{점}$ 감점**: 온도가 쾌적 구간($21.0^\circ\text{C} \sim 24.5^\circ\text{C}$) 내부인 경우
    *   **$-5\text{점}$ 감점**: 온도가 $21.0^\circ\text{C}$ 미만(서늘함) 또는 $24.5^\circ\text{C}$ 초과(더움)인 경우
*   **도입 배경**: 고온이나 저온 상태는 학습 집중도 저하 및 졸음을 유발하므로, 인간공학적 쾌적도를 반영하여 추천율을 떨어뜨리도록 가중치 밸런스를 설계했습니다.

---

## 5. 최종 점수 보정 (Capping)
*   **수식**: 
    $$SCORE_{\text{final}} = \max(10, \min(98, SCORE))$$
*   **의의**: 모든 연산 후 최종 적합도 점수가 $100\%$ 혹은 $0\%$ 같은 다소 비현실적인 수치로 표기되는 것을 막고, 사용자에게 현실적인 신뢰도를 체감하게 하고자 점수를 $10\% \sim 98\%$ 구간으로 최종 바인딩합니다.
